# Pullminder Registry

Official open-source registry of Pullminder detection and policy packs. These packs define the rules, patterns, and policies that Pullminder uses to analyze pull requests for security issues, code quality problems, and policy violations.

## Quick Start

Install a pack using the Pullminder CLI:

```bash
# Install a single pack
pullminder init --pack secrets

# Install multiple packs
pullminder init --pack secrets --pack go-security --pack bot-detection

# Install all default packs
pullminder init --defaults
```

Packs are downloaded from this registry and stored in your repository's `.pullminder/` directory.

## Available Packs

| Slug | Name | Description | Default |
|------|------|-------------|---------|
| `secrets` | Secrets Detection | Detects leaked API keys, tokens, passwords, and credentials in code and configuration files | Yes |
| `go-security` | Go Security | Identifies security vulnerabilities in Go code including SQL injection, command injection, and unsafe pointer usage | No |
| `python-security` | Python Security | Detects security issues in Python code such as unsafe deserialization, eval usage, and SQL injection | No |
| `rust-security` | Rust Security | Finds unsafe blocks, memory safety issues, and common security pitfalls in Rust code | No |
| `ruby-security` | Ruby Security | Identifies security vulnerabilities in Ruby and Rails code including mass assignment and command injection | No |
| `php-security` | PHP Security | Detects security issues in PHP code such as SQL injection, file inclusion, and deserialization flaws | No |
| `react-security` | React Security | Finds XSS vectors, unsafe innerHTML usage, and insecure patterns in React and JSX code | No |
| `infra-security` | Infrastructure Security | Detects misconfigurations in Dockerfiles, Kubernetes manifests, Terraform, and CI/CD pipelines | Yes |
| `bot-detection` | Bot Detection | Identifies PRs likely authored by bots or automated tools based on commit patterns and metadata | Yes |
| `ai-detection` | AI Detection | Detects code likely generated by AI assistants based on common patterns and markers | Yes |
| `dependency-detection` | Dependency Detection | Flags dependency changes in lockfiles and manifest files for closer review | Yes |
| `test-conventions` | Test Conventions | Enforces test naming, coverage thresholds, and test file organization standards | Yes |
| `review-quality` | Review Quality | Ensures PR descriptions, commit messages, and review comments meet quality standards | Yes |
| `sensitive-paths` | Sensitive Paths | Flags changes to security-critical paths such as auth, crypto, permissions, and infrastructure files | Yes |

## Pack Structure

Each pack lives in its own directory under `packs/` and contains a `pack.yaml` file that defines the pack metadata, patterns, and scoring rules. See [CONTRIBUTING.md](CONTRIBUTING.md) for the full pack authoring guide.
